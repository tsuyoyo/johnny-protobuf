    Login and Signup ¶

    Signup  ¶

    Signup new user
    POST/signup

    Clients send token which is got from Firebase by verifying the user in other services (e.g. Twitter).

    When the user has already been registered, API returns 200 with existing Player data.

    401 is returned, when the token cannot be authorized.

    Content-Type header Request Response
    application/protobuf PostSignupRequest PostSignupResponse

    Example URI

    POST /signup
    Response  200
    HideShow
    Headers
    Content-Type: application/json
    Response  401

    Player summaries  ¶

    List of player summaries.

    Get players' summary
    GET/player{?prefecture_id,city_id,instrument_id,studio_id}

    Various conditions can be specified in request. When multiple paramters are passed, filters work as “AND”.

    Content-Type header Request Response
    application/protobuf - GetPlayerResponse

    Example URI

    GET /player?prefecture_id=13&city_id=12345&instrument_id=1&studio_id=10
    URI Parameters
    HideShow
    prefecture_id
    number (optional) Example: 13

    filter players by prefecture

    city_id
    string (optional) Example: 12345

    filter players by city

    instrument_id
    number (optional) Example: 1

    filter players by instrument

    studio_id
    number (optional) Example: 10

    filter players by studio
